# План модернизации контент-генератора «Админ.Ко» v5.0 (Методика 2026)

## 🎯 Глобальная цель
Трансформировать текущий генератор в систему создания экспертных лонгридов-лендингов, которые доминируют в поиске за счет глубокой экспертизы, LSI-семантики и идеальной технической структуры.

---

## 🛠 Технический стек
- **Источник данных:** `real_keywords.json` (Wordstat), `SITE.md` (Конституция), `DESIGN.md` (Стиль).
- **Движок:** Gemini 1.5 Flash-Lite / Pro (через Node.js).
- **Формат:** Чистый HTML + Tailwind CSS (Stitch System).
- **Локализация:** Глубокая привязка к Кемерово (улицы, ориентиры, конкуренты).

---

## 📅 Этапы реализации

### Этап 1: Semantic & Research (Сбор данных)
- [ ] **LSI-экстрактор:** Доработка скрипта для вытягивания "хвостов" и смежных запросов из Wordstat для каждой темы.
- [ ] **Pain Point Analysis:** Определение конкретных "болей" клиента (например, не просто "ремонт PS5", а "шумит как пылесос", "вылетает в играх").
- [ ] **Equipment Integration:** Создание базы данных инструментов (Rigol, JBC, Quick) для обязательного упоминания в тексте.

### Этап 2: Multi-Stage Generation (Конвейер сборки)
- [ ] **Pipeline "Step-by-Step":**
    1. **Plan:** Генерация структуры статьи (H1 + 5-7 глав H2).
    2. **Chapters:** Генерация каждой главы отдельным запросом (для объема и глубины).
    3. **Metadata:** Генерация SEO Title и Description по формуле (Боль + Решение + Кемерово).
- [ ] **Validation Layer:** Автоматическая проверка каждого блока на наличие Markdown-мусора (`**`, `##`) и его очистка перед вставкой.

### Этап 3: Content Engineering (Экспертность и Стиль)
- [ ] **Senior Engineer Persona:** Обновление системного промпта. Запрет на "воду", фокус на технических деталях микро-пайки и диагностики.
- [ ] **Local SEO Injection:** Вплетение упоминаний Кемерово, ул. Дзержинского 2Б и специфики Кузбасса в каждую главу.
- [ ] **LSI & Formatting:**
    - Использование тегов `<b>` и `<strong>` для выделения LSI-ключей (Wordstat).
    - Категорический запрет на Markdown (`**`, `##`) — только чистый HTML.
- [ ] **Rich Elements:**
    - Генерация адаптивных HTML-таблиц цен на основе контекста услуги.
    - Блоки "Инженерный секрет" (bg-blue-50) и "Симптомы / Решение".
    - Блок FAQ с JSON-LD разметкой (Schema.org) в конце каждой статьи.
- [ ] **Smart Internal Linking:**
    - Автоматическая вставка ссылок на соответствующие CORE-лендинги (`phones.html`, `business.html` и др.).
    - Блок "Читайте также" со ссылками на 2-3 релевантные статьи блога.

### Этап 4: Structural Integrity (Техническое совершенство)
- [ ] **Auto-Footer Safeguard:** Гарантированная вставка футера и закрывающих тегов при любом сбое API.
- [ ] **FAQ JSON-LD:** Генерация микроразметки для поисковых систем внутри статьи.
- [ ] **Image Placeholders:** Подготовка адаптивных контейнеров под будущие реальные фото ремонта.

### Этап 5: Реновация Блога (Mass Upgrade)
- [ ] **Cleanup:** Удаление дублей и битых файлов.
- [ ] **Total Regeneration:** Поэтапная перегенерация всех 196 статей по новому стандарту.

---

## 📈 Критерии успеха (Definition of Done)
1. Каждая статья имеет объем 8 000 - 15 000 знаков.
2. Статья содержит таблицу цен, FAQ и экспертный блок.
3. В тексте упомянуто профессиональное оборудование (Rigol/JBC).
4. Отсутствуют обрывы кода (битые футеры).
5. Статья проходит проверку на наличие LSI-ключей из Wordstat.
