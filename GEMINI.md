# 📜 КОНСТИТУЦИЯ ПРОЕКТА «АДМИН.КО»

Это FOUNDATIONAL MANDATE. Любой агент ИИ или скрипт обязан следовать этим правилам. Нарушение ведет к деградации продукта.

---

## 1. АРХИТЕКТУРНАЯ ИЗОЛЯЦИЯ
Проект разделен на два независимых модуля. Изменения в одном НЕ должны влиять на другой.

*   **CORE (Лендинги):** Файлы в корне `site/public/`. Здесь важна структура секций.
*   **BLOG (Статьи):** Файлы в `site/public/blog/`. Здесь важна дизайнерская верстка текста (`prose`) и вложенные элементы конверсии.

### Правило синхронизации UI:
Любой скрипт синхронизации (например, `sync-ui.js`) обязан использовать **разную логику** для CORE и BLOG. 
*   **В Блоге запрещено менять что-либо между концом `<nav>` и началом `<footer>`.** 
*   Разрешена только «хирургическая» замена навигации и футера по точным тегам.

---

## 2. ДИЗАЙН-СИСТЕМА (STITCH)
Запрещено заменять уникальные дизайнерские блоки на стандартные шаблоны.

### Ключевые компоненты:
1.  **Блок «Спросите инженера»:** Обязан сохранять оригинальную верстку (карточки `bg-gray-50` или `bg-blue-50`, скругления `rounded-3xl`, тени `shadow-sm`).
2.  **Кнопки:** Использовать только системные классы (`bg-primary`, `hover:bg-primary-dark`). Категорически запрещено оставлять «белые кнопки на белом фоне».
3.  **Цвета:** Использовать переменные Tailwind из `tailwind.config.js` (`bg-background-light`, `text-text-dark`).

---

## 3. ПРАВИЛА ГЕНЕРАЦИИ КОНТЕНТА (AI)
Скрипты генерации (`auto-blogger.js` и др.) должны работать в режиме «Silent Expert».

1.  **Output:** Только чистый HTML. Никаких «Вот ваш текст», «Я очистил статью», «Готово».
2.  **No Markdown:** В итоговые файлы не должны попадать решетки (`##`) или звездочки (`**`). Только валидные HTML-теги.
3.  **SEO Integrity:** Мета-теги `title` и `description` обязательны и должны соответствовать `index.json`.

---

## 4. ПРАВИЛА РАБОТЫ С ФАЙЛАМИ
1.  **Full Content Preservation:** Категорически запрещено использовать комментарии типа `<!-- Остальной контент... -->` при записи файлов. Файл должен перезаписываться целиком с сохранением всех данных.
2.  **Git Safety:** Перед массовыми изменениями всегда создавать бэкап-ветку.
3.  **Validation:** После любого изменения `header.html` или `footer.html` запускать `node scripts/sync-ui.js` и проверять визуально и CORE, и BLOG.
