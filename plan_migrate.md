# 📜 ТЕХНИЧЕСКИЙ РЕГЛАМЕНТ МОДЕРНИЗАЦИИ КОНТЕНТ-СИСТЕМЫ «АДМИН.КО»
## Версия: 5.5 (Март 2026) | Стек: Gemini 2.5 + Headless WP

Этот документ является ОФИЦИАЛЬНЫМ ПЛАНОМ перехода от статической генерации HTML к динамической системе управления контентом на базе WordPress с сохранением высокой скорости работы сайта.

---

## 1. ЦЕЛЕВАЯ АРХИТЕКТУРА (2026)

Система работает как цепочка Node.js скриптов без использования n8n:
1.  **SEO Анализ:** `ai-keyword-analyzer.js` (Gemini 2.5 Flash + Search).
2.  **Генератор тем:** `generate-topics.js` (Формирует `prioritized_topics.json`).
3.  **Оркестратор:** `content-orchestrator.js` (Управляет циклом).
4.  **Генератор контента:** `auto-blogger-module.js` (Gemini 2.5 Pro + Nano Banana Image Engine).
5.  **Публикатор:** `wp-publisher.js` (WP REST API).

---

## 2. ТЕХНОЛОГИЧЕСКИЙ СТЭК (МАРТ 2026)

### А. Модели Gemini 2.5
*   **gemini-2.5-pro:** Основная модель для написания лонгридов (3000+ слов). 
    *   *Функция:* Reasoning (мышление) и Grounding (заземление в поиске Google).
    *   *Инструмент:* `google_search` (нативный поиск цен, новостей и тех. бюллетеней за 2026 год).
*   **gemini-2.5-flash-image (Nano Banana Pro):** Мультимодальная модель генерации.
    *   *Функция:* Создание технических иллюстраций, обложек и инфографик в 4K.
    *   *Синтаксис:* `responseModalities: ["IMAGE"]`.

### Б. Backend (CMS)
*   **WordPress 6.x (Headless):** Используется как хранилище данных.
*   **Auth:** Application Passwords (Base64 Auth).
*   **Plugins:** 
    *   *Rank Math SEO PRO:* Управление мета-тегами и Schema.org через API.
    *   *WPGraphQL:* (Опционально) для быстрой выборки данных.

## 3. ПОШАГОВЫЙ ПЛАН АДАПТАЦИИ СКРИПТОВ

### Шаг 1: Адаптация SEO-анализа (`ai-keyword-analyzer.js`)
*   Внедрить `google_search` в промпт анализатора.
*   Цель: Находить не только высокочастотные ключи, но и «горячие» инфоповоды марта 2026 (например, «массовый брак шлейфов iPhone 17 Pro»).

### Шаг 2: Модернизация Генератора (`auto-blogger-module.js`)
*   **Объем:** Увеличить количество секций до 12-15.
*   **Ресерч:** Добавить этап сбора фактов перед написанием (Gemini Pro собирает JSON с данными по Кемерово).
*   **Контент:** 
    *   `Blitz-Answer` — блок с ценой и сроком в первом экране.
    *   `Key Takeaways` — выводы.
    *   `Technical Deep Dive` — детальное описание оборудования (JBC, JCID).
*   **Визуал:** Скрипт генерирует промпт для `gemini-2.5-flash-image`, получает изображение и передает его в `wp-publisher.js`.

### Шаг 3: Реализация Публикатора (`wp-publisher.js`)
*   Метод `uploadMedia()`: Загрузка сгенерированного изображения в медиабиблиотеку WP.
*   Метод `publishPost()`: Отправка JSON-пакета (Title, Content, Excerpt, Featured Image ID, Rank Math Meta).

### Шаг 4: Синхронизация Фронтенда
*   Создать скрипт `scripts/wp-to-static.js`.
*   Функция: Раз в час опрашивает WordPress API, забирает новые статьи и рендерит их в `.html` файлы по вашим текущим шаблонам Tailwind.
*   *Результат:* Сайт остается сверхбыстрым статическим ресурсом, но контент управляется через удобную админку WP.

## 4. СТРУКТУРА СТАТЬИ 2026 (EEAT + SEO)
